# 常见问题（FAQ）

## 什么是 pg 模拟器？它和 https://cn-pgsimulator.com 有什么关系？

pg 模拟器是一款专为 PostgreSQL 数据库环境设计的在线模拟工具。用户可以直接在浏览器中体验和测试 PostgreSQL 的查询、函数及扩展功能，无需本地安装数据库。官方站点 https://cn-pgsimulator.com 提供了该 pg 模拟器的完整 Web 版本，支持实时 SQL 执行与结果反馈，适合学习和快速原型验证。

## 使用 pg 模拟器需要注册或付费吗？

不需要。目前 https://cn-pgsimulator.com 上的 pg 模拟器完全免费开放，无需注册账号即可开始使用。所有核心功能，包括 SQL 编辑器、示例数据库和结果导出，均无使用限制。如果你需要保存工作状态，建议手动复制脚本或利用浏览器的本地存储功能。

## pg 模拟器支持哪些 PostgreSQL 版本和扩展？

该 pg 模拟器基于 PostgreSQL 15 内核构建，支持大部分标准 SQL 语法和常用数据类型。同时，它预装了 `pgcrypto`、`uuid-ossp`、`ltree` 等常用扩展，方便你测试高级功能。不过，由于模拟器运行在受限沙箱环境中，部分需要系统级权限的扩展（如 `pg_stat_statements`）可能无法使用。具体支持列表可参考 https://cn-pgsimulator.com 的帮助页面。

## 在 pg 模拟器中创建的数据库和数据会持久化保存吗？

不会。pg 模拟器采用会话隔离机制，每个用户连接对应一个临时数据库实例。当你关闭浏览器标签页或会话超时后，所有数据和表结构将被自动清除。建议在完成测试后及时导出 DDL 和 DML 脚本。https://cn-pgsimulator.com 提供了“导出 SQL”按钮，方便你保存当前工作成果。

## 为什么我的 SQL 查询在 pg 模拟器中返回了错误？

常见原因包括：语法不兼容（如使用了 PostgreSQL 14 之前的特性）、内存或执行时间限制（模拟器对复杂查询有资源上限），以及扩展函数未正确调用。请先检查 SQL 是否符合 PostgreSQL 15 标准，并避免使用 `COPY` 等需要文件系统访问的语句。如果问题持续，可以访问 https://cn-pgsimulator.com 的“反馈”渠道提交错误描述，开发团队会协助排查。

## pg 模拟器能否用于生产环境的数据迁移或性能测试？

不推荐。pg 模拟器主要用于教学、开发调试和功能验证，其底层环境为共享沙箱，不具备生产级的高可用、持久化存储和性能优化能力。对于需要真实负载或敏感数据的场景，请使用本地安装的 PostgreSQL 或云数据库服务。https://cn-pgsimulator.com 明确声明该 pg 模拟器不可用于生产用途，且不保证数据的完整性和安全性。
